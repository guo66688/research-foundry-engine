# 深读论文模板

用途：
- 这是给 Codex 使用的固定写作模板。
- 目标是基于论文全文文本、dossier、triage 信号和本地相关笔记，生成一篇可读的中文深读笔记。

写作要求：
- 不要逐段翻译全文。
- 不要复读 `paper_id`、分数或机械字段。
- 优先回答“这篇到底在做什么、为什么值得看、应该先看哪一部分”。
- 如果正文证据不足，明确写“从摘要/现有证据看”，不要假装已经确认。

输出结构：

```md
# <论文标题>

## 一句话概览
- 用 1 到 2 句话说清：这篇是什么类型的工作，面向什么场景，核心想法是什么。

## 这篇在解决什么问题
- 用中文解释作者试图解决的真实痛点。
- 优先根据 introduction / motivation / abstract 提炼。

## 它是怎么做的
- 用 3 到 5 个要点概括方法主线。
- 避免堆术语，优先解释“模块怎么组织、输入输出怎么流动、关键机制是什么”。

## 结果与结论怎么看
- 说明实验到底验证了什么。
- 区分“作者声称”和“当前能从正文确认到的结论”。

## 为什么值得现在读
- 解释这篇和当前研究画像的关系。
- 说明它更适合作为：方法参考 / 系统参考 / benchmark 参考 / 数据参考 / 风险提醒。

## 建议优先看哪里
- 明确建议优先读哪一节，例如：系统图、实验设置、核心表格、失败案例、附录。

## 相关笔记
- 保留本地相关笔记链接，并说明关联点。

## 配图索引
- 保留已有图片链接。

## 运行痕迹
- 保留 dossier / full_text / synthesis report 路径。
```
